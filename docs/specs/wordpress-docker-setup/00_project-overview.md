# プロジェクト概要: WordPress Docker環境構築

## spec名
`wordpress-docker-setup`

## タスクの概要
Docker ComposeでWordPress開発環境を構築し、ローカル開発に適した設定を行う。

## タスクの目的
- ローカル開発環境をDockerで再現可能にする
- チーム間で統一された開発環境を提供する
- 環境構築の手間を最小化する
- データの永続化とポータビリティを確保する

## スコープ

### 含まれる内容
- Docker Composeによる複数コンテナのオーケストレーション
- WordPressコンテナの構成
- MySQL/MariaDBデータベースコンテナの構成
- ローカル開発に適した設定（デバッグモード、ボリュームマウント等）
- データ永続化のためのボリューム設定
- 環境変数による設定の外部化

### 含まれない内容
- 本番環境用の設定
- SSL/HTTPS設定
- メール送信設定
- WordPress本体のカスタマイズ
- テーマやプラグインの開発

## 前提条件
- Dockerがインストール済みであること
- Docker Composeがインストール済みであること
- 基本的なDockerとWordPressの知識があること

## 制約事項
- ローカル開発環境のみを対象とする
- ポート競合を避けるため、既存の開発サーバーとの調整が必要な場合がある

## 期待される成果物
- docker-compose.yml（または docker-compose.yaml）
- 環境変数設定ファイル（.env.example等）
- データ永続化のためのボリューム設定
- 起動・停止手順のドキュメント（README等）

## 技術スタック
- Docker
- Docker Compose
- WordPress（最新安定版またはLTSバージョン）
- MySQL/MariaDB

## 次フェーズへの引き継ぎ事項

### 要件フェーズで明確化すべき項目
1. **WordPressのバージョン**
   - 最新版、特定のバージョン、どちらを使用するか

2. **データベースの選択**
   - MySQL か MariaDB、どちらを使用するか
   - バージョン指定の必要性

3. **ポート設定**
   - WordPressのアクセスポート（デフォルト: 8000または80）
   - データベースのポート公開の必要性

4. **ボリュームマウント対象**
   - wp-content全体 または themes/plugins のみ
   - データベースデータの永続化方法

5. **開発支援機能**
   - WP_DEBUGの有効化
   - エラーログの出力設定
   - その他の開発用設定

6. **パフォーマンス設定**
   - メモリ制限
   - アップロードファイルサイズ制限

7. **phpMyAdminの必要性**
   - データベース管理用UIの追加要否
